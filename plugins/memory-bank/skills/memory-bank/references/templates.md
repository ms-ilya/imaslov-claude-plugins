## Writing Guidelines

1. **Be concrete, not vague.** "Handles OAuth 2.0 refresh via `TokenManager.refresh()`" — not "manages authentication."
2. **Use verified paths.** Every file path must pass the verification check.
3. **Keep it scannable.** Tables > paragraphs. Short bullets > long sentences.
4. **No history.** Never write "we changed X to Y." Just describe Y.
5. **No speculation without flags.** If uncertain, use `⚠️ INFERRED` with evidence of what WAS verified.
6. **Feature folders use kebab-case:** `user-profile/`, `push-notifications/`.
7. **Cross-reference with relative paths:** `../other-feature/IMPLEMENTATION.md`.
8. **ACTIVE_CONTEXT.md speculation:** In-Progress Work and Next Steps are inherently speculative. Use `⚠️ Ask user` placeholders for fields that cannot be determined from code. Mark all inferred claims `⚠️ INFERRED` until user confirms.
