# Add Decision to Memory Bank

You are recording a technical decision for a feature in the project's Memory Bank.

---

## Parse Arguments

**Expected format:** `<feature-name>: <decision description>`

Example: `/add-decision authentication: Use Keychain instead of UserDefaults for token storage`

1. **Feature name** — everything before the first `:`
2. **Decision description** — everything after the `:`

If format is unclear, ask the user to clarify which feature and what was decided.

## Steps

1. Check if `memory-bank/<feature-name>/` exists
   - If not → ask if the user wants to create it first with `/update-memory-bank <feature>`
2. Check if `memory-bank/<feature-name>/DECISIONS.md` exists
   - If not → create it using the DECISIONS.md template:

```markdown
<!-- Generated by memory-bank -->

# [Feature Name] — Decisions

> User-curated only. Add decisions via: `/add-decision feature: description`

## Decisions
```

3. Add a new decision entry at the **top** (newest first) using this format:

```markdown
---

### [Decision Title]

- **Status:** Accepted
- **Context:** Why this decision was needed
- **Decision:** What was decided
- **Rationale:** Why this option was chosen
- **Alternatives Rejected:**
  - Option: Why not
- **Consequences:** Trade-offs and implications
```

4. Ask the user for any missing fields: rationale, alternatives considered, consequences
5. **Never overwrite existing decisions.** Append only. Mark superseded ones as "Superseded by [link]" if applicable.
